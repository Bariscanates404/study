1 Summary of the logic
 2   patterns:
 3    a: dpm (delete-person) mutation flow
 4    b: pid (:person/id) flow
 5    c: ppr (person props) flow
 6    d: lid (:list/id) flow
 7   a: dpm (delete-person) mutation flow
 8    a1: dpm definition
 9    a2: passed to transact! function
10    a3: which is assigned to local variable in parent component PersonList 
11    a4: which is set to :onDelete parameter of child component Person 
12    a5: Person component receives it as second props
13    a6: Person component sets it to :onClick attribute of dom/button
14   b: pid (:person/id) flow
15    b0: Person component sets pid parameter as argument of onDelete function that is set to :onClick attribute of dom/button
16    b1: On runtime, user clicks button. pid is passed to onDelete function
17    b2: delete-person closure function receives pid
18    b3: It passes pid to dpm
19    b4: dpm receives pid argument
20    b5: dpm passes pid to remove-ident* function which is called by swap! function
21    note: remove-ident* uses pid to find out which Person ident should be removed from the list of idents.
22   c: ppr (person props) flow
23    c1: PersonList calls map with Person component factory and people vector that contains person props
24    c2: map passes ppr one-by-one to the comp/computed closure function inside Person component factory function 
25    c3: Person component receives ppr
26   d: lid (:list/id) flow
27    d1: PersonList component receives lid 
28    d2: It passes lid to dpm
29    d3: dpm receives lid
30    d4: dpm passes lid to remove-ident* function which is called by swap! function
31    note: remove-ident* uses lid to find out PersonList that you want to remove Person ident from
